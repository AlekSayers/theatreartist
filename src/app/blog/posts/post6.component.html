The Demo

Let's say you wanted to create a simple shopping list app. You want to have three pages to the app, showing "to do", "in progress", and "completed" tasks. Here are some very basic requirements:

  1. Each page must load tasks from a data source and display them
  2. Tasks that are "to do" must be able to be advanced to "in progress"
  3. Tasks that are "in progress" must be able to be advanced to "completed"

You'll notice that most of the logic across each page is similar:

  • Each page requires a call to a data source for similar data
  • Each page should be able to refresh to bring in fresh data
  • The "to do" and "in progress" pages both require the ability to update a task's status

Model

Let's start with the simplest component of MVC: the Model. In web applications, the model is almost always some kind of data source, like a database. Angular generally recommends that the model be accessed and manipulated through services (see https://docs.angularjs.org/guide/services for more details). In the below example, TaskService represents the model, where the only data of concern is an Array of tasks. Depending on the state of the application, a subarray of these tasks would be retrieved based on their type.

Controller

The responsibility of the controller is to define business logic. Simplicity is key here; delegate significant logic to services. Angular provides excellent documentation about how to use controllers effectively (see https://docs.angularjs.org/guide/controller).

View

Binding your Angular code to the view is simple. In your index.html file:

  1. Import AngularJS
  2. Import your controller code
  3. Import your model code
  4. Bootstrap your code using angular.bootstrap
